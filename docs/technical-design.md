# 技术设计与 AI 接口说明

## 架构

- React + TypeScript + vinext/Vite：界面和应用构建。
- Three.js + React Three Fiber：三维桥梁、航线、航点、视锥与无人机。
- 本地 TypeScript 教学数据：四类航线、约 2 m 采样航点与病害空间关联。
- Canvas 图像层：原图、分割轮廓、掩膜、标尺和最大宽度标注。

## 三维坐标约定

桥梁沿 X 轴布置，总长 180 m；Y 轴为高度；Z 轴为横桥向。桥面下缘约 5.3 m，教学标称净空约 5 m。桥面航线位于桥面上方约 10 m，两侧航线距桥边约 5 m，桥底航线距梁底约 2 m并在桥墩位置折线绕避。经纬度由本地坐标基于固定教学基准点换算，并明确标为教学模拟坐标。

## AI 适配层

当前 MVP 使用 `lib/teaching-data.ts` 中的教学演示结果完成全流程。接入真实模型时，建议新增 `lib/ai-adapter.ts`，统一返回：裂缝编号、分割掩膜、多段线、最大宽度、长度、置信度、风险等级、构件编号、桥梁局部三维坐标和原始图像 URL。

浏览器端可使用 ONNX Runtime Web 加载 U-Net 或 YOLO 分割模型；生产部署也可将图像提交至后端推理服务。无论采用哪种方式，返回数据均应映射为 `CrackRecord`，三维与界面层无需改变。

## 离线性

桥梁几何、地面、草地、航线和无人机均由程序生成；裂缝巡检图像位于 `public/inspection/`，运行时不依赖地图、字体、模型或贴图 CDN。
